# 通用定义和函数

## 1. 基本数据类型
### 1.1. 单一事件 （遥信）
`tSingleEvent` 通常用于表示一个单一的事件，例如某个遥信（DI）的状态变化。

`tSingleEvent` 数据结构主要用于以下场景：
1. **遥信状态变化报告：**
    - 当某个遥信点的状态发生变化时，RTU 会生成一个 tSingleEvent 结构的数据，并通过 IEC 60870-5-101 或 IEC 60870-5-104 协议将其发送给主站。主站接收到这个事件后，可以及时更新状态并采取相应的措施。
2. **事件记录：**
    - tSingleEvent 可以用于记录和存储事件，以便后续的分析和审计。例如，记录某个开关的合闸和分闸事件。
3. **状态监视：**
    - 主站可以通过定期查询或接收 tSingleEvent 数据，监视系统的实时状态，确保系统的正常运行。
```c
typedef uint8_t tSingleEvent;

typedef tSingleEvent* SingleEvent;

void
SingleEvent_setEventState(SingleEvent self, EventState eventState);

EventState
SingleEvent_getEventState(SingleEvent self);

/* 设置基本数据的品质描述 QDP */
void
SingleEvent_setQDP(SingleEvent self, QualityDescriptorP qdp);

/* 查看基本数据的品质描述 QDP */
QualityDescriptorP
SingleEvent_getQDP(SingleEvent self);
```

### 1.2. 状态和状态变化检测
`StatusAndStatusChangeDetection` 是 lib60870 中用于表示状态和状态变化检测的数据结构。这个数据结构通常用于 IEC 60870-5-101 和 IEC 60870-5-104 协议中的遥信（DI）和遥测（AI）数据传输。

它不仅包含当前的状态值，还包含一个标志位，用于指示状态是否发生了变化。

StatusAndStatusChangeDetection 数据结构的主要作用包括：
- **状态报告：**
    用于报告当前的状态值。例如，遥信点的开/关状态。

- **状态变化检测：**
    用于检测和报告状态的变化。这对于实时监控和故障检测非常重要。例如，当某个开关从“开”变为“关”时，可以及时通知主站。

- **优化数据传输：**
    通过 changed 标志位，可以减少不必要的数据传输。只有在状态发生变化时，才发送数据，从而节省带宽和资源。

```c
typedef struct sStatusAndStatusChangeDetection tStatusAndStatusChangeDetection;

typedef tStatusAndStatusChangeDetection* StatusAndStatusChangeDetection;

/* 其中：
 * encodedValue[0] 和 encodedValue[1] 表示 ST ，即状态值 
 * encodedValue[2] 和 encodedValue[3] 表示 CD ，即状态变化标志位
 */
struct sStatusAndStatusChangeDetection {
    uint8_t encodedValue[4];
};

/* 获取状态值，即全部 16 位 */
uint16_t
StatusAndStatusChangeDetection_getSTn(StatusAndStatusChangeDetection self);

/* 获取状态变化标志位，即全部 16 位 */
uint16_t
StatusAndStatusChangeDetection_getCDn(StatusAndStatusChangeDetection self);

/* 设置状态值，即全部 16 位 */
void
StatusAndStatusChangeDetection_setSTn(StatusAndStatusChangeDetection self, uint16_t value);

/* 获取某一位的状态值，index不大于16 */
bool
StatusAndStatusChangeDetection_getST(StatusAndStatusChangeDetection self, int index);

/* 获取某一位的状态变化标志位，index不大于16 */
bool
StatusAndStatusChangeDetection_getCD(StatusAndStatusChangeDetection self, int index);
```

## 3. 文件相关定义和函数
### 3.1. FileReady
表示文件传输准备就绪的信号。通常用于告知接收方文件传输即将开始。

### 3.2. SectionReady
表示文件部分（section）传输准备就绪的信号。通常用于告知接收方文件部分传输即将开始。

### 3.3. FileCallOrSelect
用于请求文件或文件部分的传输。通常用于发起文件传输请求。

### 3.4. FileLastSegmentOrSection
表示文件或文件部分的最后一个段（segment）或部分（section）传输完成。

### 3.5. FileACK
表示文件传输的确认消息。通常用于确认接收到的文件或文件部分。

### 3.6. FileSegment
表示文件的一部分（段）。通常用于传输文件的各个部分。

### 3.7. FileDirectory
表示文件目录。通常用于传输文件系统的目录结构。

### 3.8. QueryLog
表示日志查询请求。通常用于请求特定的日志数据。
